# calendar — 일정 취합 · 약속 잡기

여러 사람의 일정을 모아 **약속 잡기 좋은 날짜**를 추천하는 웹앱입니다.
빌드 과정 없는 정적 사이트(HTML + Vue 3 CDN + Firebase)로 동작합니다.

## 기능

1. **일정 등록** — Google 로그인 후 날짜·시간(또는 종일)으로 내 일정을 등록.
   - 같은 날짜에 여러 사람이 각자 일정을 등록할 수 있음.
2. **소유권** — 내 일정은 나만(또는 공유 링크 보유자만) 수정/삭제 가능. 다른 로그인 사용자는 내 일정을 건드릴 수 없음.
3. **공유 링크** — 일정마다 `공유링크` 버튼으로 토큰 링크 생성. 링크 보유자는 로그인 없이(익명 인증) 또는 다른 계정으로 해당 일정만 편집 가능.
4. **시간 단위 등록** — 시작/종료 시간 지정 또는 종일.
5. **최적 일정 추천** — 참가자들의 일정을 취합해 약속 가능한 날짜를 점수순으로 추천.
   - **엄격 모드**(기본): 그 날 일정이 하나라도 있는 사람이 있으면 그 날짜 제외.
   - **시간대 고려 모드**: 모두의 빈 시간대를 계산해 모임 소요시간이 들어가는 공통 슬롯이 있으면 후보로.
   - **가산점**: 다음날이 공휴일(+3) / 다음날이 주말(+2) / 당일이 휴일(+1).

## 실행

Google 로그인 팝업은 `localhost` 도메인에서 동작합니다(Firebase 기본 허용 도메인).

```bash
cd calendar
python3 -m http.server 8000   # 또는: npx serve .
# 브라우저에서 http://localhost:8000 접속
```

## Firebase

- travel 프로젝트와 동일한 Firebase 프로젝트(`hosing-5913f`)를 사용합니다.
- 데이터는 Realtime Database의 `calendar/schedules` 경로에 저장됩니다.
- **필요한 보안 규칙 (Firebase 콘솔):** `calendar` 경로에 인증 사용자 읽기/쓰기 허용. 예:
  ```json
  { "rules": { "calendar": { ".read": "auth != null", ".write": "auth != null" } } }
  ```
- 공유 링크의 비로그인 편집을 쓰려면 콘솔에서 **익명 인증**을 활성화하세요. 비활성 시 Google 로그인으로 안내됩니다.

## 한계 / 참고
