# calendar — 모임 일정 취합 · 약속 잡기

여러 사람의 **불가한 날짜**를 모아 모두가 가능한 날짜를 추천해 주는 웹앱입니다.
빌드 과정 없는 정적 사이트(HTML + Vue 3 CDN + Firebase)로 동작합니다.

## 핵심 흐름

1. **관리자**가 홈에서 새 모임을 만듭니다. (구글 로그인 불필요)
2. 만들어진 모임 페이지의 **공유 링크**를 참여자들에게 전달합니다.
3. **참여자**는 링크로 접속해서 자기 이름을 입력한 뒤,
   달력에서 **참석이 어려운 날짜를 드래그/클릭**해 등록합니다.
4. **관리자**는 참여자들이 등록한 불가 일정을 바탕으로
   "**가능 날짜 추천**" 화면에서 모두가 가능한 날짜(+공휴일 가산점)를 순위로 확인합니다.

## 주요 기능

- **모임(그룹) 단위 관리** — URL의 `?g={groupId}` 로 모임을 구분. 관리자는 여러 모임을 만들 수 있음.
- **로그인 없음** — Firebase 익명 인증을 사용. 사용자는 별명만 입력하면 됨.
- **역할 구분**
  - 관리자: 모임을 만든 사람(브라우저에 저장된 키로 인식). 모임 이름 변경/삭제, 모든 참여자 일정 조회·수정·삭제, 가능 날짜 추천.
  - 참여자: 공유 링크로 접속한 사람. 자기가 등록한 불가 일정만 수정/삭제 가능.
- **달력 중심 UI** — 큰 월간 달력에서:
  - **날짜 클릭** → 그날 상세(누가 불가인지) 조회
  - **날짜 드래그** → 여러 날짜 범위로 불가 일정 등록
- **불가 일정 등록** — 종일/시간 지정, 사유(선택), 메모(선택)
- **가능 날짜 추천 (관리자 전용)**
  - 지정한 기간에서 모두가(또는 최소 N명이) 가능한 날짜를 점수순으로 정렬
  - 가산점: 다음날 공휴일(+3) / 다음날 주말(+2) / 당일 휴일(+1) / 전원 가능(+1)
  - 추천된 날짜는 달력에서 ⭐ 표시로 강조

## 실행

정적 파일이라 임의의 정적 서버로 서비스하면 됩니다.

```bash
cd calendar
python3 -m http.server 8000   # 또는: npx serve .
# 브라우저에서 http://localhost:8000 접속
```

## Firebase 설정

- Realtime Database의 `calendar/groups/{groupId}/...` 경로에 데이터가 저장됩니다.
- **익명 인증**을 반드시 활성화해야 합니다. (Firebase 콘솔 → Authentication → 로그인 방법 → 익명 사용 설정)
- 보안 규칙 예시 (`calendar` 경로에 인증 사용자 읽기/쓰기 허용):
  ```json
  { "rules": { "calendar": { ".read": "auth != null", ".write": "auth != null" } } }
  ```

## 데이터 구조

```
calendar/
  groups/
    {groupId}/
      meta:
        id, name, creatorUid, creatorKey, creatorName, createdAt
      schedules/
        {scheduleId}:
          id, title, dateStart, dateEnd,
          allDay, start, end, memo,
          ownerUid, ownerKey, ownerName,
          createdAt, updatedAt
```

- `creatorKey` / `ownerKey`는 브라우저 localStorage에 저장된 랜덤 키입니다.
  모임 생성 브라우저 = 관리자로 인식됩니다.
  브라우저 데이터가 지워지면 관리자 권한을 잃을 수 있으니 주의하세요.

## 한계 / 참고

- 공휴일 데이터는 `holidays.js`에 하드코딩되어 있습니다. 필요하면 직접 추가/수정하세요.
